## 2. 인터랙션 개념

```
사용자가 텍스트 입력 중
        ↓
커서를 원하는 위치에 둔 채로
        ↓
키보드 상단 버튼 클릭 (맞춤법 / 문맥수정 / 답변생성)
        ↓
커서 앞 텍스트를 컨텍스트로 AI 처리
        ↓
결과 제안 → 사용자가 선택 → 텍스트필드에 삽입 (전송 아님)
```

**핵심 규칙**
- **커서 위치 기준**으로 앞의 텍스트를 읽어 AI가 처리한다
- 읽어오는 텍스트는 **1문단씩 처리**한다 (ENTER 기준으로 단락 분리)
- 결과 선택 시 **텍스트필드에 내용이 삽입**되며, 전송은 사용자가 직접 한다

```javascript
// 커서 기준 현재 문단 추출
const textarea = document.getElementById('input');
const cursorPos = textarea.selectionStart;
const fullText = textarea.value.substring(0, cursorPos);
const paragraphs = fullText.split('\n');
const currentParagraph = paragraphs[paragraphs.length - 1];
```

## 3. 핵심 기능 명세

### 기능 1 · 맞춤법 + 문맥수정

> 버튼은 분리되어 있고, 클릭한 버튼의 결과가 결과 패널 최상단에 표시된다.  
> 내부적으로는 **하나의 AI 호출**로 두 결과를 동시에 받아온다.

**트리거**
- `맞춤법` 클릭 → 맞춤법 결과 카드 상단, 문맥수정 카드 하단 (흐리게)
- `문맥수정` 클릭 → 문맥수정 결과 카드 상단, 맞춤법 카드 하단 (흐리게)

**입력**
- 커서 기준 현재 문단 텍스트

**처리 (AI)**
- 맞춤법·띄어쓰기 오류 감지 및 교정
- 문맥상 어색한 표현 감지 및 제안

**출력 (JSON)**
```json
{
  "spelling": {
    "corrected": "교정된 전체 문장",
    "changes": [
      { "original": "낳아?", "fixed": "나아?", "reason": "맞춤법 오류" }
    ]
  },
  "context": {
    "has_suggestion": true,
    "suggestion": "내일 만날 장소 정하는 게 좋을 것 같아, 어디가 편해?",
    "reason": "더 자연스러운 질문 형태"
  }
}
```

**결과 카드 UI**
- `적용` 클릭 → 현재 문단을 교정된 텍스트로 교체 후 텍스트필드에 반영
- `닫기` 클릭 → 결과 패널 닫기
- 문맥 제안이 없으면 (`has_suggestion: false`) 문맥 카드 숨김

**Claude 프롬프트**
```
당신은 한국어 맞춤법 및 문맥 교정 전문가입니다.
아래 텍스트의 맞춤법·띄어쓰기를 교정하고,
문맥상 어색한 부분이 있으면 제안해주세요.
반드시 JSON 형식으로만 출력하세요.

텍스트: {current_paragraph}
```

### 기능 2 · 답변생성

**트리거**: 커서를 두고 `답변생성` 버튼 클릭

**입력**
- 커서 기준 현재 문단 텍스트 (상대방 메시지로 간주)

#### 2-1. 말투 CHIP (3종 고정 + 커스텀 롤)

| CHIP | 설명 |
|------|------|
| 친근 | 친구에게 말하듯 자연스럽고 가벼운 말투 |
| 공손 | 비즈니스·교수님·선배 등 격식체 |
| 사용자가 만든 커스텀 롤 | 저장된 롤 순서대로 CHIP 추가 |
| + 추가 | 새 커스텀 롤 생성 화면 진입 (항상 마지막) |

> **CHIP 정렬 로직**: AI가 상대 메시지의 맥락(격식/비격식, 관계, 톤)을 판단해 `suggested_order` 기준으로 자동 정렬. `+ 추가`는 항상 마지막.

#### 2-2. 선택 이력 축적 (개인화)

사용자가 답변 카드를 선택하면 해당 이력을 localStorage에 저장.  
다음 답변 생성 시 이 이력을 프롬프트에 포함해 사용자 취향에 맞는 답변을 우선 생성.

```javascript
// 선택 시 이력 저장
const history = JSON.parse(localStorage.getItem('replyHistory') || '[]');
history.push({
  receivedMessage: currentParagraph,
  selectedTone: tone,
  selectedText: text,
  timestamp: Date.now()
});
// 최근 20개만 유지
if (history.length > 20) history.shift();
localStorage.setItem('replyHistory', JSON.stringify(history));
```

**프롬프트에 이력 반영**
```
[사용자 과거 선택 패턴]
{recent_history_summary}
위 패턴을 참고해 사용자 취향에 맞는 말투를 우선적으로 추천하세요.
```

**선택 시 동작**
- 카드 클릭 → 텍스트필드에 해당 답변 삽입
- 전송은 사용자가 직접 수행 (자동 전송 없음)
- 선택 이력 localStorage에 저장

#### 2-3. 커스텀 롤 생성

사용자가 직접 말투 롤을 만들어 저장. 만들어둔 롤은 CHIP 목록에 자동 추가됨.

**입력 폼 (플레이스홀더로 예시 표시)**

| 필드 | 플레이스홀더 예시 |
|------|-----------------|
| 이름 | `예: 교수님, 직장 상사, 선배님` |
| 말투 가이드 | `예: 격식체, ~습니다 체, 존댓말 필수. 업무적이고 간결하게.` |

```html
<input type="text" placeholder="예: 교수님, 직장 상사, 선배님" />
<textarea placeholder="예: 격식체, ~습니다 체, 존댓말 필수. 업무적이고 간결하게."></textarea>
```

**저장 방식**: localStorage에 JSON 배열로 저장
```json
[
  { "id": 1, "name": "교수님", "guide": "격식체, ~습니다 체, 존댓말 필수." },
  { "id": 2, "name": "직장 상사", "guide": "업무적이고 간결하게, 결론 먼저." }
]
```

**Claude 프롬프트 (답변생성)**
```
당신은 한국어 커뮤니케이션 전문가입니다.
아래 메시지에 대한 답변을 말투별로 생성해주세요.

[말투 목록]
- 친근: 친구에게 말하듯 자연스럽고 가벼운 말투
- 공손: 격식체, 존댓말 (~습니다, ~요)
{커스텀 롤 목록}

[사용자 과거 선택 패턴]
{recent_history_summary}

상대 메시지의 맥락을 분석해 각 말투별 답변을 생성하고,
가장 적절한 말투 순서를 suggested_order 배열로 알려주세요.
반드시 JSON 형식으로만 출력하세요.

받은 메시지: {current_paragraph}
```

**출력 (JSON)**
```json
{
  "suggested_order": ["공손", "친근", "교수님"],
  "replies": [
    { "tone": "친근", "text": "응 나 시간 돼! 어디서 먹을까?" },
    { "tone": "공손", "text": "네, 내일 점심 괜찮을 것 같습니다." },
    { "tone": "교수님", "text": "네, 내일 말씀하신 시간에 맞춰 뵙겠습니다." }
  ]
}
```

## 4. 시스템 아키텍처 (MVP)

```
┌─────────────────────────────────────────┐
│           로컬 웹앱 (index.html)          │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │         텍스트 입력 영역          │    │
│  └─────────────────────────────────┘    │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  [CHIP: 공손][친근][교수님][+추가]│  ← 답변생성 시 노출
│  │  AI 결과 카드 (맞춤법/문맥/답변)  │    │
│  └─────────────────────────────────┘    │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  [✦맞춤법][↺문맥수정][✉답변생성]│  ← B스타일 pill 버튼
│  ├─────────────────────────────────┤    │
│  │         키보드 UI 모킹            │    │
│  └─────────────────────────────────┘    │
└──────────────────┬──────────────────────┘
                   │
          ┌────────┴────────┐
          ▼                 ▼
   Claude API          localStorage
   (AI 처리)       (롤 저장 + 선택 이력)
```

## 5. 기술 스택

| 구분 | 선택 | 이유 |
|------|------|------|
| Frontend | 단일 index.html (HTML + CSS + JS) | 로컬 실행, 별도 빌드 불필요 |
| 실행 | 로컬 브라우저 직접 열기 or `python -m http.server` | 설치 없이 즉시 실행 |
| API Key | 앱 내 입력창 UI로 주입 | 하드코딩 없이 안전하게 |
| AI | Claude API (claude-sonnet-4-5-20251001) | 한국어 성능, 빠른 응답 |
| 저장소 | localStorage | 커스텀 롤 + 선택 이력 저장 |

## 7. 디자인 가이드

### 7-1. AI 기능 버튼 (확정: B안)

| 버튼 | 아이콘 | 비활성 | 활성 |
|------|--------|--------|------|
| 맞춤법 | A자 교정 SVG | 투명 배경 + 테두리 | 검정 배경 + 흰 글씨 |
| 문맥수정 | 줄+화살표 SVG | 투명 배경 + 테두리 | 검정 배경 + 흰 글씨 |
| 답변생성 | 메시지 버블 SVG | 투명 배경 + 테두리 | 검정 배경 + 흰 글씨 |

```css
.ai-btn {
  display: flex;
  align-items: center;
  justify-content: center;
  gap: 5px;
  padding: 6px 0;
  border-radius: 100px;
  border: 1.5px solid #C0C0C0;
  font-size: 11.5px;
  font-weight: 500;
  flex: 1;
  white-space: nowrap;
  min-width: 0;        /* 답변생성 잘림 방지 */
}
.ai-btn svg { width: 13px; height: 13px; flex-shrink: 0; }
.ai-btn.on  { background: #1a1a1a; color: white; border-color: #1a1a1a; }
```

### 7-2. 결과 카드

- 활성: `border: 1.5px solid #1a1a1a`
- 비활성: `opacity: 0.5`
- 맞춤법 태그: `#E8F5E9 / #2E7D32`
- 문맥수정 태그: `#FFF8E1 / #E65100`

### 7-3. CHIP

```css
.chip     { font-size:11px; padding:3px 10px; border-radius:20px; border:0.5px solid #ccc; white-space:nowrap; }
.chip.on  { background:#1a1a1a; color:white; }
```

## 10. 시연 시나리오 (1분, 노트북 기준)

1. **(10초)** 앱 소개 — "키보드 버튼 하나로 AI를, 앱 전환 없이"
2. **(15초)** 맞춤법 → 문맥수정 탭 전환 시연
3. **(15초)** 답변생성 — CHIP 전환하며 말투별 답변 확인 후 선택 → 텍스트필드 삽입
4. **(10초)** 커스텀 롤 추가 — "직장 상사" 롤 만들고 바로 적용
5. **(10초)** 마무리 — "모든 앱에서, 키보드 하나로"
